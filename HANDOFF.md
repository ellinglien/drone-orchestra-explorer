## What This Is

An immersive 3D audio-visual web experience for browsing the Drone Orchestra community archive. Users select a month from a timeline, then enter a spatial audio environment where each orbiting entity represents one community member's drone submission.

**Live Experience Flow:**
1. User visits `drones.rpmchallenge.com` (future subdomain)
2. Dark overlay appears with chronological month list
3. User clicks "March 2026 - Theme: Chaos"
4. Month selector fades away
5. 3D space loads with orbiting drone entities
6. Each entity plays its audio spatially as user navigates with WASD + mouse
7. User clicks "◄ Change Month" to explore other months

## Architecture Overview

### High-Level System

```
┌─────────────────────────────────────────┐
│  Google Drive                           │
│  /Drone Orchestra/                      │
│    ├── 2025-01/                         │
│    ├── 2025-04 Chaos/                   │
│    └── 2026-01/                         │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  GitHub Action (daily 6am UTC)          │
│  1. Fetch folder list from Drive        │
│  2. Download audio files                │
│  3. Convert to MP3 VBR (FFmpeg)         │
│  4. Upload to DO Spaces CDN             │
│  5. Generate public/data/months.json    │
│  6. Commit to repo                      │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  Vercel (auto-deploy on push)           │
│  - Serves static public/ directory      │
│  - Loads months.json on page load       │
│  - Streams audio from DO Spaces CDN     │
└─────────────────────────────────────────┘
```

### Tech Stack

**Backend (GitHub Actions):**
- Node.js 18+
- Google Drive API v3
- FFmpeg (audio conversion)
- AWS S3 SDK (for DO Spaces)

**Frontend:**
- A-Frame 1.2.0 (WebGL/VR framework)
- Vanilla JavaScript (no build system)
- Handlebars templates (pre-rendered to static HTML)

**Infrastructure:**
- GitHub Actions (free for public repos)
- Vercel (static hosting)
- DigitalOcean Spaces (CDN, shared with RPM site)

## Data Flow Detail

### months.json Structure

Generated by GitHub Action, consumed by frontend:

```json
{
  "generated": "2026-04-04T06:00:00Z",
  "months": [
    {
      "id": "2026-04",
      "name": "April 2026",
      "theme": "Chaos",
      "droneCount": 12,
      "drones": [
        {
          "artist": "USTAKNOW",
          "url": "https://rpm3.nyc3.cdn.digitaloceanspaces.com/drone-orchestra/2026-04/ustaknow-chaos-drone.mp3",
          "originalFilename": "USTAKNOW - Chaos Drone.ogg"
        }
      ]
    }
  ]
}
```

**Artist name extraction patterns:**
- `"Artist - Title.ext"` → `"Artist"`
- `"Artist_Title.ext"` → `"Artist"`
- `"NoSeparator.ext"` → `"NoSeparator"`

**Month folder patterns:**
- `"2026-04"` → April 2026 (no theme)
- `"2026-04 Chaos"` → April 2026, theme: Chaos
- `"2026-05 D Minor"` → May 2026, theme: D Minor

## What's Complete

### ✅ Backend (100%)

1. **Google Drive Integration**
   - Fetches folder list from parent folder
   - Downloads audio files (WAV, FLAC, OGG, MP3, M4A)
   - Parses artist names from filenames

2. **Audio Processing**
   - Converts all formats to MP3 VBR (quality 4, ~165kbps)
   - Handles batch conversion with temp file cleanup
   - Safari/iOS compatible (OGG not supported there)

3. **CDN Upload**
   - Uploads to DigitalOcean Spaces
   - Public-read ACL for browser streaming
   - Path: `drone-orchestra/{YYYY-MM}/{filename}.mp3`

4. **Metadata Generation**
   - Creates months.json with structured data
   - Sorts newest first
   - Includes themes parsed from folder names

5. **GitHub Actions Workflow**
   - Scheduled daily at 6am UTC
   - Manual trigger for testing
   - Commits months.json to repo
   - Triggers Vercel auto-deploy
   - Includes timeout, concurrency control, error summaries

### ✅ Frontend (100%)

1. **Month Selector UI**
   - Dark overlay with chronological month list
   - Highlights latest month
   - Shows drone count per month
   - Smooth fade transitions

2. **3D Scene Controller**
   - Dynamically loads drones from months.json
   - Creates A-Frame audio assets and entities
   - Staggered fade-in animation (200ms delay between drones)
   - Preserves existing orbital-motion physics

3. **Metadata Display**
   - Month info overlay (top-right): name, theme, count
   - "◄ Change Month" button (top-left)
   - Navigation flow between selector and scene

4. **Visual Design**
   - Dark brutalist aesthetic preserved
   - Red/orange color scheme (#ff4500)
   - High contrast, minimal UI
   - Monospace typography

5. **Vercel Configuration**
   - Static build system (Handlebars → HTML)
   - Cache headers (5min for JSON, 1yr for audio)
   - Security headers (nosniff, frame-deny)

## Integration with RPM Website

### Option 1: Subdomain (Recommended)

**Setup:**
- Deploy as `drones.rpmchallenge.com`
- Add link in RPM site navigation
- Completely separate application (no code integration needed)

**RPM Website Changes:**
```html
<!-- Add to navigation -->
<a href="https://drones.rpmchallenge.com">Drone Orchestra Archive</a>
```

**Pros:**
- Zero code integration
- Independent deployment cycle
- No impact on RPM site performance
- Easy to maintain

**Cons:**
- Separate domain to manage

### Option 2: Iframe Embed

**Setup:**
- Deploy to Vercel subdomain or custom domain
- Embed in RPM site page

**RPM Website Changes:**
```html
<!-- New page: /drone-orchestra -->
<iframe
  src="https://drone-orchestra.vercel.app"
  width="100%"
  height="800px"
  frameborder="0"
  allow="autoplay"
></iframe>
```

**Pros:**
- Single domain (rpmchallenge.com/drone-orchestra)
- Integrated navigation

**Cons:**
- Autoplay permissions complex in iframe
- Performance overhead
- Harder to debug

### Option 3: Full Integration (Not Recommended)

**Setup:**
- Copy public/ files into RPM site
- Integrate JavaScript into RPM site build

**Cons:**
- Heavy lifting for integration
- Two build systems to maintain
- A-Frame conflicts with existing site
- Not worth the effort

**Recommendation:** Use **Option 1 (Subdomain)** - clean, simple, maintainable.

## Maintenance

- Check GitHub Action logs (Actions tab)
- Verify Vercel build logs
- Confirm secrets are configured
